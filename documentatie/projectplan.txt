WAT IS DIT PROJECT?
--------------------------------------------------------------------------------
BeleggenCoach is een AI-beleggingscoach voor Belgische beginners zonder
beleggingservaring. Doel: ETF-keuze begrijpbaar maken, een persoonlijk plan
opstellen en gedragscoaching bieden op eigen tempo.

JURIDISCH: dit is GEEN gereguleerd financieel advies (MiFID II). Puur educatie,
vergelijking en simulatie. Elke pagina met ETF-informatie draagt een disclaimer.


ARCHITECTUURVISIE (DDD + HEXAGONAAL)
--------------------------------------------------------------------------------
De app volgt Domain-Driven Design met hexagonale lagen:

  Domeinlaag     -- pure Python/TS logic, geen infra-imports
       omhoog
  Applicatielaag -- use cases, event-orchestratie, MCP-aanroepen
       omhoog
  Infrastructuurlaag -- DB (SQLAlchemy), Redis, externe APIs, MCP clients
       omhoog
  Interface-laag -- FastAPI routers, Next.js pages

MCP servers zijn Open Host Services: publieke contracten voor Copilot agents.
Externe integraties (Keycloak (self-hosted), OpenAI, yfinance) zitten achter een ACL.

TECH STACK
--------------------------------------------------------------------------------

  Frontend   : Next.js 15, React 19, TypeScript strict, Tailwind CSS
  Backend    : Python 3.12, FastAPI, SQLAlchemy async, Alembic, Pydantic v2
  Database   : PostgreSQL 16 (Docker), Redis (cache + chat memory)
  MCP        : Python mcp SDK, stdio protocol -- output altijd { success, data, error }
  Auth       : Keycloak (self-hosted, optioneel, conditioneel via env var)
  AI         : OpenAI GPT-4o-mini (chat, met mock fallback)
  Marktdata  : yfinance ACL adapter (gepland)
  Monorepo   : pnpm workspaces (frontend) + uv (Python)

DATAFLOWS
--------------------------------------------------------------------------------

Onboarding (doelarchitectuur):
  1. Frontend wizard  -->  POST /api/v1/onboarding/start    (user aanmaken)
  2.                  -->  PUT  /api/v1/onboarding/{id}/profile  (profiel)
  3. Backend valideert + slaat op in PostgreSQL
  4. investor-profile-mcp  leest profiel -> berekent RiskScore value object
  5. etf-data-mcp:get_top3_for_profile  -> top 3 ETFs
  6. portfolio-plan-mcp    -> genereert eerste plan
  7. Frontend toont op dashboard

AI Chat (doelarchitectuur):
  1. Frontend POST /api/v1/chat/message  {user_id, message}
  2. FastAPI haalt user profiel uit PostgreSQL
  3. Redis: laad chat history (max 20 msgs)
  4. etf-data-mcp:get_top3_for_profile -> systeemcontext injecteren
  5. OpenAI GPT-4o-mini stream response
  6. Redis: sla nieuw bericht op
  7. SSE stream terug naar frontend

Check-in + Behavior Coaching:
  1. Frontend POST /checkins  {invested, emotional_state, notes}
  2. Event: CheckinRecorded
  3. behavior-coach-mcp:detect_fomo_pattern  -> analyse
  4. behavior-coach-mcp:monthly_checkin_summary -> coach_response
  5. Opslaan in DB, terugsturen naar frontend


KWALITEITSPOORTEN (architect-niveau)
--------------------------------------------------------------------------------
  - Geen directe verwijzingen tussen aggregates (alleen via ID)
  - Domeinlaag heeft GEEN imports uit infrastructuurlaag
  - Elke externe integratie (Keycloak (self-hosted), OpenAI, yfinance) zit achter een ACL
  - Ubiquitous language consistent in code EN documentatie
  - Alle MCP tools hebben JSON Schema op input EN output
  - Scores altijd in mensentaal, nooit als kaal getal
  - Geen "koop dit nu" taal -- altijd "past mogelijk bij jouw profiel omdat..."
